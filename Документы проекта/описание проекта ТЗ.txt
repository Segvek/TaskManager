========================================== Идея ============================================
	Основная идея Task Manager это составление регулярного, ежедневного плана для достижения поставленых целей и контролировать даний процесс, путём выполнения таких шагов:
	-  Постановка цели, задачи или определение своей "мечты";
	-  Определение текущей обстановки (финансовых и временних ресурсов);
	-  Разбиение глобальной задаи на несколько более лёгких подзадач;
	-  Ежедневное составление списка задач которые необходимо выполнить на следующий день и выполнение задач поставлених в предыдущий день;
	-  Промежуточный анализ выполнения задач (достижения цели). 
	Посколько для достижения любой цели необходимы некоторые ресурсы в основном это деньги и времья, а даный проект уже поможет решить проблему времени. Возникает необходимость создать интегрированый сервис для слежения за финансами в разрезе достижения поставлених целей.  
	Сервис слежения за финансами предполагает фиксирования промежуточных результатов разных счетов. К примеру у меня есть несколько счетов: наличные, банковская карта, мне должны, я должн и цель №1. На сервесе можно будет настроить уведомления пользователя, к при меру раз в неделю, что неохоимо пересчитать средства и обновить дынные. Таким образом пользователь сможет следить за общей динамикой своих средств. Также счёт можно привязать к цели, таким образом отображение строки о финансовых ресурсых может подсвечиватся разными цветами или просто отображать сстояние при наличии или отсутствии денежных средств на счёте.
	Ресурс помимо отслеживания состояний счетов также будет предоставлять возможность отслеживание трат на поставленую цель (предпологаемые и фактические затраты). 
	Пользователь должен иметь возможность оценивать и сохранять оценки (процентовки) достижения поставленой цели.

------------------------------------Продукт веб-сайт----------------------------------------
	Перечень функций предоставляемый клиентам разрабатываемого ПО:
	1)  Создание целей, а также их редактирование(предупреджение о том что редактирование цели свидетельствует о том что человек дал слабину) и "удаление";
	2)  Создание плана на завтра (функция доступна только после указаного времени-вечером)  
	3)  Создание, редактирование, удаление счетов
	4)  Фиксирование состояний счетов 
	5)  Подтвердить выполнения пункта ежедневного плана
	6)  Просмотр графика активности
	7)  Просмотр отчёта по конкретной цели 
	8)  Просмотр отчета по конкретному пункту глобального плана
	9)  Просмотр отчёта о сосотоянии своих средств 
	10) Фиксирование фактических затрат на выполнение пункта глобального плана (п.г.п.) 
	11) фиксирование процента выполнения п.г.п. 
	12) Просмотр фактических затрат на достижения поставленой цели
	13) Настройка уведомлений

	ДЕТАЛЬНОЕ ОПИСАНИЕ ФУНКЦИЙ 
	1) При создании цели человек должен опредилить следующие моменты: название цели, краткое описание, сроки и капитал для выполнения поставленой цели, составить план с 10-15 пунктов (каждый пункт плана имеет свои сроки а также капитал для его выполнения) для достжения цели. Текже при создании цели человек имеет возможность сразу определить для неё прибыточный счёт (деталнее смотри пункт 1.а).
	Помимо выше описаных значений в план будут входить другие составляющие которые описаны в пункте 1.б.

	1.а) Счёт это некий реальный или виртуальный кошелёк который может бить либо прибыточным либо убыточным. Счёт может иметь подсчёта одного уровня вложености. Счёт имеет название и динамику движения средств с момента регистрации пользователя. 
	Для того что бы изменить состояние какого-либо счёта необходимо создать новый срез (промежуточный релультат) и внести данные о сотояниях всех счетов.

	1.б) Глобальный план для достижения поствленой цели
	К каждому пункту плана может быть привязан один счёт и етот счёт будет разсматриваться как независимый.
	Каждый пункт плана имеет оценку (процент) выполнения даного пункта. 
	Можно смотреть отчёт по каждому пункту плана, данный отчёт будет состоять из отчёта о фактических затратах и перечень действий выполненых для его выполнения. Перечень действий будет состоять из ежедневных задач детальнее смотри пункт 2.

	2) План на завтра создается вечером (необходимо настроить время уведомления на телефон)  и его можно редактировать только до 23:59, после чего план нельзя редактировать или удалить. Каждый пункт плана имеет один из трёх статусов (не выполнен, начат, выполнен) и предположительное время выполнения. По каждому пункту плана может быть настроено уведомление пользователя. Также любой пункт плана может быть прикреплён к пункту глобального плана достижения цели. 
	
	3) Создание счетов происходит как в независимой форме к примеру новая карта для зарплаты так и зависимо от цели, данный вариант создания счёта возможет только из формы просмотра цели и только в том случае если счёт для цели ещё не создан. Под редактированием имеется ввиду редактрование названия счета. При удалении счета удаляется вся информация связана с фиксацией и соответственно будет изменено состояние суммы в отчёте о состоянии своих средств. При удалении счёта привязаному к цели происходит отвьязка счёта от цели. 
	
	4) Даный пункт предпологает что человек в произвольный момент или по напоминанию зашел на свою персональную страницу и инициировал процес фиксирования состояний счетов. При фиксации необходимо заполнить значения всех счетов и подтвердить фиксацию система автоматически запишет дату и время фмксации. 

	5) Для подтверждения выполнения необходимо открыть "задачи на сегодня" и там изменить статус. 
	
	6) Просмотр графика активности предпологает что человек может посмотреть график за месяц который будет содержать колличество поставленых задач, колличесво выполненых задач на день. Даний отчёт состовляется на основании ежедневных планов. (В перспективе можно добавить графики поставленых и выполненых задач привязаных к какой-либо цели);

	7) Отчёт по конкретной цели будет в себе содержать список задач выполнених на протяжении указаного времени для достежения даной цели. Даный список формируется исходя из привязки п.п. из ежедневного плана. Также даный отчёт будет содержать график отображающий предпологаемые и фактические затраты на даную цель

	8) данный отчёт будет состоять из отчёта о фактических затратах на даный п.г.п. и перечень действий выполненых для его выполнения. Перечень действий будет состоять из ежедневных задач детальнее смотри пункт 2.
		
	9) Просмотр отчёта о состоянии своих средст предпологает что человек формирует отчёт в виде таблици в которой указано состояния счетов при каждой фиксации за конкретный промежуток и сумарное финансовое состояние. (В дальнейшем можно данный отчет предоставлять в нескольких формах к примеру в виде графиков)
